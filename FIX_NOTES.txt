Scout Backend v3.4 VERIFIER FIXED

What changed from v3.3:
1. Added built-in email verification endpoints.
2. Added GET /verifier-config.
3. Added POST /verify-email.
4. Added POST /batch-verify-emails.
5. Added POST /verify-dork-leads to verify saved dork leads in the backend.
6. Added POST /import-verification-results for importing results from external email verifiers.
7. Added GET /verified-leads and /verified-leads?status=ready&format=csv.
8. Dork leads now carry verificationStatus, verificationScore, verificationProvider, verificationReason, verifiedAt, and readyToContact.

Built-in verification mode:
- Works without paid API key.
- Checks email format, disposable domains, role-based emails, free-provider emails, and DNS/MX records.
- It does NOT falsely claim mailbox-level validity. Without a verifier API key, deliverable domains are marked needs_provider or risky.

Optional mailbox-level verifier providers:
Set EMAIL_VERIFIER_PROVIDER to one of: zerobounce, abstract, hunter, neverbounce, kickbox.
Then add the matching API key env var:
- ZEROBOUNCE_API_KEY
- ABSTRACT_EMAIL_API_KEY
- HUNTER_API_KEY
- NEVERBOUNCE_API_KEY
- KICKBOX_API_KEY

Recommended for Airtable outreach:
1. Collect leads with signals.
2. Run POST /verify-dork-leads.
3. Export only /verified-leads?status=ready&format=csv.
4. Send outreach only to readyToContact leads.
